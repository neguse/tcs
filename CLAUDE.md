# TinyC# (tcs) — C#サブセット → Lua 5.5 トランスパイラ

## プロジェクト概要

C#の極小サブセットをフロントエンド言語とし、Lua 5.5ソースコードをバックエンドとするトランスパイラ。
詳細は `objective.md` を参照。

## ビルド・テスト

### 前提

- .NET 10 SDK / C# 14
- Lua 5.5 (`deps/lua/lua`)

### コマンド

```bash
# テスト実行（基本操作）
dotnet test

# トランスパイラ実行
dotnet run --project Transpiler -- input.cs -o output.lua

# Lua 5.5 ビルド
make -C deps/lua

# Lua でトランスパイル結果を実行
deps/lua/lua output.lua

# 全テスト（シェルスクリプト）
bash run-tests.sh
```

## 開発ワークフロー

### 作業手順

1. `doc/current.md` で現在の状態を確認
2. `doc/tasks.md` から次のタスクを選択
3. テストを先に書く（TDD）
4. 実装する
5. `dotnet test` で全テスト通過を確認
6. `doc/done.md` に完了ログを記録
7. `doc/current.md` を更新
8. コミット（1タスク = 1コミット）

### 完了ログ形式（done.md）

```markdown
### タスク名 ✓ (YYYY-MM-DD)
- [実施内容]
- [変更ファイル]
- よかったこと: [成功したアプローチ、再利用可能なパターン]
- 判断: [トレードオフ、選択理由、却下した代替案]
- 残課題: [残作業]
```

### コミットルール

- 1タスク = 1コミット（`git bisect` / `git revert` 可能に）
- テスト + 実装 + ドキュメント更新をセットで
- コミットメッセージ: `feat: T{番号} {概要}`（例: `feat: T3 if/else文のトランスパイル`）

## コード規約

### C# (.NET 10 / C# 14)

- `<TargetFramework>net10.0</TargetFramework>` / `<LangVersion>14</LangVersion>`
- nullable enable（全プロジェクト）
- ファイルサイズ上限: 600行（警告）、800行（禁止）
- テストは xUnit + FluentAssertions スタイル
- Roslyn の `Microsoft.CodeAnalysis.CSharp` パッケージを使用

### Lua 出力

- インデント: スペース2つ
- Lua 5.5 互換のみ（5.4以前の機能に依存しない）
- 生成コードにはヘッダコメントを付与: `-- Generated by TinyC# transpiler`

### テスト方針

- **セマンティックテスト**: C#入力 → トランスパイル → Lua VM実行 → 結果検証
- 出力文字列の完全一致テストは使わない（Lua出力形式の変更に弱い）
- `TranspileAndRun(csharpSource, luaExpr)` ヘルパーで統一
- Lua 5.5 バイナリ (`deps/lua/lua`) をプロセス起動して実行

## トランスパイル方針

### 基本マッピング

| C#                  | Lua 5.5                            |
|---------------------|------------------------------------|
| class               | table + metatable                  |
| struct              | 初期は非対応（classに寄せる）       |
| enum                | integer 定数テーブル                |
| method              | function(self, ...)                |
| static method       | function(...)                      |
| property            | getter/setter メソッド              |
| field               | table フィールド                    |
| List\<T\>           | sequence table `{v1, v2, ...}`     |
| Dictionary\<K,V\>   | hash table `{[k]=v, ...}`         |
| lambda              | function(...) ... end              |
| interface           | 型チェックのみ（Lua出力なし）       |
| namespace           | ネストされた table モジュール        |
| string interpolation| string.format                      |

### Roslyn活用

- `CSharpCompilation` でセマンティック解析
- `IOperation` (Bound Tree) を走査してLuaコード生成
- 型情報は Roslyn から取得、自前の型システムは持たない

## lub3d API interface 生成（TinyCSharpGen）

lub3d の Generator パイプラインに乗せて C# interface を自動生成する。

### アーキテクチャ

```
Clang AST → TypeRegistry → ModuleSpec（既存IR）
                                ├→ CBindingGen.Generate()   → .c  (既存)
                                ├→ LuaCatsGen.Generate()    → .lua (既存)
                                └→ TinyCSharpGen.Generate() → .cs  (新規)
```

### BindingType → C# 型マッピング

| BindingType       | C# 型           |
|-------------------|-----------------|
| Int               | int             |
| Int64             | long            |
| UInt32            | uint            |
| Float             | float           |
| Double            | double          |
| Bool              | bool            |
| Str               | string          |
| Void              | void            |
| Struct(_, _, lua) | class名         |
| Enum(_, lua)      | enum名          |
| Callback(p, r)    | Action/Func     |
| Vec2              | Vector2         |
| Vec4              | Vector4         |
| Ptr(inner)        | inner（透過）    |
| VoidPtr           | nint            |

### 生成物の配置

- lub3d の `Generator/TinyCSharp/TinyCSharpGen.cs` に実装
- 出力先: tcs の `TinySystem/` または `gen/` ディレクトリ
- lub3d の `Program.cs` に `TinyCSharpGen.Generate(spec)` 呼び出しを追加

## 既知の制約・注意事項

- dotnet と Lua の数値精度差異（double vs Lua number）
- 文字列: C# は UTF-16、Lua は バイト列（UTF-8前提）
- struct: 初期は非対応（classに寄せる）。C API最適化フェーズで再検討
- null vs nil: C# の null は Lua の nil にマップ
- 型消去: ジェネリクスはコンパイル時のみ、Lua出力には型情報なし
